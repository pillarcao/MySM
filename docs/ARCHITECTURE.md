### 前端组件交互

```
App.vue (状态管理 + 路由)
  │
  ├─→ LoginView ─────── POST /api/auth/login ──→ JWT Token → localStorage
  │
  ├─→ 菜单导航 ─────── GET /api/meta/tables ──→ 按 MENU_GROUP 分组渲染
  │
  └─→ DynamicTableManager (每个 Tab 一个实例)
        │
        ├─→ 加载元数据 ─── GET /api/meta/{tableId} ──→ 字段定义、树配置
        ├─→ 加载列表 ──── GET /api/dynamic/{tableId}/list?status=EDIT
        ├─→ 加载下钻 ──── GET /api/meta/drills/{tableId}
        │
        ├─→ RecordList (左面板)
        │     ├─→ 树形数据 ── GET /api/meta/{tableId}/tree
        │     └─→ 点击节点 ── 筛选中间表格
        │
        ├─→ 中间表格
        │     ├─→ 行选中 ─── 更新右面板
        │     ├─→ 右键 ──── ContextMenu
        │     └─→ 编辑 ──── inline editing
        │
        └─→ RecordDetail (右面板)
```

## 五、核心数据流

### 5.1 记录生命周期 (状态流转)

```
                         ┌─────────────────┐
                         │   NEW / UPDATE   │
                         │  (REL_FLG=N,     │
                         │   COMP_FLG=N)    │
                         └────────┬────────┘
                                  │
                            SAVE (可多次)
                                  │
                                  ▼
                         ┌─────────────────┐
              ┌─────────│     EDIT 态      │─────────┐
              │          │  (N, N)          │         │
              │          └────────┬────────┘         │
              │                   │                   │
         ROLLBACK            EDITCOMP              DELETE
         (D→B恢复)          (标记完成)            (需先EDITCOMP)
              │                   │                   │
              │                   ▼                   ▼
              │          ┌─────────────────┐    ┌──────────┐
              │          │  EDITCOMP 态    │    │  已删除   │
              │          │  (N, Y)          │    └──────────┘
              │          └────────┬────────┘
              │                   │
              │              RELEASE
              │           (B表→D表同步)
              │                   │
              │                   ▼
              │          ┌─────────────────┐
              └──────────│  RELEASE 态     │
                         │  (Y, Y)          │
                         │  + D表已同步      │
                         └─────────────────┘
```

### 5.2 Save 操作流程

```
前端 Save 按钮
      │
      ▼
POST /api/dynamic/{tableId}/save
      │
      ▼
┌─────────────────────────┐
│  ValidationService      │
│  执行 SM_CHECK_DEF      │
│  (NOTNULL, LOCK_FREE,   │
│   KEY, STAT_N...)        │
└────────────┬────────────┘
             │ 通过
             ▼
┌─────────────────────────┐
│  DynamicCrudService     │
│  ├─ 判断 Insert/Update  │
│  ├─ 构造动态 SQL        │
│  ├─ REL_FLG 纳入 WHERE  │
│  └─ 执行 JdbcTemplate   │
└────────────┬────────────┘
             │
             ▼
┌─────────────────────────┐
│  HistoryService         │
│  5层 shift-register     │
│  LAST_*1 ← 当前操作     │
│  LAST_*2 ← 原 LAST_*1  │
│  LAST_*3 ← 原 LAST_*2  │
│  ...                    │
└────────────┬────────────┘
             │
             ▼
        返回成功
```

### 5.3 Release 操作流程

```
前端 Release 按钮
      │
      ▼
POST /api/release/{tableId}
      │
      ▼
┌─────────────────────────────────────────┐
│  ValidationService                      │
│  执行 RELEASE 类型校验                    │
│  (NOTNULL, STAT_N, LOCK_FREE...)        │
└────────────────────┬────────────────────┘
                     │ 通过
                     ▼
┌─────────────────────────────────────────┐
│  ReleaseService.release()               │
│                                         │
│  1. 读取 B 表 REL_FLG='N' 记录          │
│  2. 提取 RELEASE_PARAM (D 表主键)       │
│  3. 提取 RELEASE_WRITE (D 表字段)       │
│  4. D 表 UPSERT:                        │
│     ├─ UPDATE → 影响行数 > 0? 完成      │
│     └─ INSERT (如果不存在)              │
│  5. 删除 B 表旧 REL_FLG='Y' 记录       │
│  6. 更新 B 表: REL_FLG='Y', COMP_FLG='Y'│
│  7. 记录操作历史                        │
│                                         │
└─────────────────────────────────────────┘
         │
         ▼
   B表(N,N) ──→ B表(Y,Y) + D表已同步
```

## 八、关键设计模式

| 模式 | 说明 |
|------|------|
| **双表策略 (B/D)** | B 表 = 工作副本(可编辑)，D 表 = 发布快照(只读) |
| **复合主键版本化** | B 表主键 = 业务键 + REL_FLG，同一记录可存在双版本 |
| **配置驱动 UI** | SM_TABLE_DEF + SM_FIELD_DEF 驱动前端渲染，零代码新增表 |
| **声明式校验** | SM_CHECK_DEF 解耦校验逻辑，无需改代码即可调整规则 |
| **Shift-Register 审计** | 5 层操作历史内嵌于记录本身，无需独立审计表 |
| **悲观锁** | LOCK_USER + LOCK_TIME 防止并发编辑冲突 |
| **动态 SQL** | JdbcTemplate 根据元数据拼接 SQL，一套代码服务 88 张表 |
| **三栏 MDI 布局** | 树/列表(左) → 表格(中) → 详情(右)，支持多表 Tab 切换 |
